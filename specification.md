# SDD Navigator Dashboard – Спецификация

## 1. Контекст
Веб-приложение для визуализации и анализа метрик покрытия спецификаций в системе разработки.
Цель — дать разработчикам и тест-менеджерам единую точку входа для оценки полноты тестирования и документирования компонентов.

## 2. Цели
1) Пользователь видит общую метрику покрытия спецификаций (%) и её историческое изменение (тренд).
2) Пользователь просматривает список всех модулей с индивидуальным покрытием каждого.
3) Пользователь кликает на модуль и видит список спецификаций этого модуля с их статусом (covered/not covered).
4) Пользователь может фильтровать модули по статусу (excellent/good/warning/critical) и искать по названию.
5) Пользователь видит детали для каждого модуля и спецификации: дата последнего обновления.

## 3. Ограничения (что делать НЕ НУЖНО)
- Редактирование спецификаций и статусов через UI (только для просмотра).
- Интеграция с внешними системами (Jira, TestRail и т.д.) — в MVP используются моковые данные.
- Экспорт отчётов (PDF, Excel).
- Историческое сравнение версий спецификаций (только текущее состояние).
- Углублённый анализ трендов и прогнозы.

## 4. Пользовательские сценарии

### S1. Главный дашборд с метриками и визуализацией
- Карточка с основной метрикой: общий процент покрытия спецификаций (например, 80.7%)
- Показатели: количество модулей, общее количество спецификаций, количество покрытых спецификаций
- **Три графика для визуализации:**
  - Линейный график: история общего покрытия за последние 7–14 дней (тренд)
  - Столбчатая диаграмма: распределение модулей по статусам (сколько модулей в each/good/warning/critical)
  - Круговая или донат-диаграмма: доля покрытых vs непокрытых спецификаций (визуальное соотношение)
- Графики интерактивны: hover-подсказки показывают точные значения

### S2. Список модулей
- Таблица или карточная сетка с модулями
- Столбцы: название модуля, процент покрытия, количество покрытых/всего спецификаций, статус
- Цветовая кодировка: зелёный (≥95%), светло-зелёный (80%-94%), жёлтый (60–79%), красный (<60%)
- Сортировка по покрытию (возрастание/убывание)
- Поиск по названию модуля (поле ввода)
- Фильтры по статусу (checkboxes): excellent, good, warning, critical
- Результаты обновляются в реальном времени при вводе поиска и изменении фильтров

### S3. Детали модуля
- Название модуля и его сводные метрики (% покрытия, количество спецификаций)
- Таблица спецификаций модуля с колонками: название, статус (covered/not covered), дата последней проверки
- Кнопка или ссылка "Назад к списку модулей"
- Макет доступен по клику на модуль из списка (S2) или через навигацию

### S4. Загрузка данных и состояние
- При открытии дашборда данные загружаются с API Routes (`/api/metrics`, `/api/modules`, `/api/modules/[id]`)
- Индикатор загрузки (spinner/skeleton) показывается во время загрузки данных
- Страница не выводит ошибки в консоли браузера

### S5. Навигация
- Главная страница дашборда доступна по маршруту `/`
- Детали модуля доступны по маршруту `/modules/[id]`
- Back-навигация работает корректно (кнопка или browser back)

## 5. Бизнес-правила
- **Покрытие = (Covered Specs / Total Specs) × 100%**, округляется до одного десятичного знака.
- **Статус модуля** определяется по его покрытию: excellent ≥95%, good 80–94%, warning 60–79%, critical <60%.
- **Дата обновления** модуля — это дата последней проверки любой из его спецификаций.
- Данные обновляются при каждой загрузке страницы (GET запрос к API).
- Тренд покрытия отражает историю за последние 7–14 дней (в MVP — сгенерированные данные).

## 7. API Контракт (высокоуровневое описание)
- **GET /api/metrics** — возвращает общие метрики (overall coverage, trend, counts)
- **GET /api/modules** — возвращает список всех модулей с их метриками
- **GET /api/modules/[id]** — возвращает детали модуля и его спецификации
- Все endpoints возвращают 200 при успехе, 404 если ресурс не найден, 500 при ошибках сервера

## 8. Качество и наблюдаемость
- Unit-тесты: валидация расчёта покрытия, определение статуса модуля
- Integration-тесты: загрузка дашборда, клик на модуль и загрузка деталей
- Переходы и навигация отработаны без ошибок
